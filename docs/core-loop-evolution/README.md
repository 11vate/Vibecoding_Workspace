# Core Loop Evolution

## Purpose

Track and design how core loops evolve over time, ensuring loops maintain engagement as users progress.

**Core loops must evolve to maintain engagement. Static loops become boring.**

---

## Evolution Dimensions

### Complexity Evolution

**How Complexity Evolves**:
- Start simple
- Add complexity gradually
- Introduce new mechanics
- Increase depth

**Why**: Gradual complexity increase maintains engagement.

---

### Depth Evolution

**How Depth Evolves**:
- Start shallow
- Add depth gradually
- Enable mastery
- Support optimization

**Why**: Gradual depth increase enables mastery.

---

### Variety Evolution

**How Variety Evolves**:
- Start focused
- Add variety gradually
- Introduce options
- Enable exploration

**Why**: Gradual variety increase maintains interest.

---

## Evolution Patterns

### Linear Evolution

**Pattern**: Steady, predictable evolution.

**Characteristics**:
- Consistent additions
- Predictable growth
- Steady complexity
- Clear progression

**Use Case**: Clear progression paths.

**Why**: Linear evolution provides clarity.

---

### Exponential Evolution

**Pattern**: Rapid, accelerating evolution.

**Characteristics**:
- Rapid additions
- Accelerating growth
- Increasing complexity
- Expanding possibilities

**Use Case**: High growth phases.

**Why**: Exponential evolution creates excitement.

---

### Branching Evolution

**Pattern**: Multiple evolution paths.

**Characteristics**:
- Multiple paths
- User choice
- Different experiences
- Replayability

**Use Case**: User agency focus.

**Why**: Branching evolution enables choice.

---

## Evolution Design

### Evolution Triggers

**Trigger Types**:
- **Milestone-Based** - After milestones
- **Time-Based** - Over time
- **Skill-Based** - As skill increases
- **Choice-Based** - User choices

**Why**: Multiple triggers enable flexible evolution.

---

### Evolution Hooks

**Hook Types**:
- **New Mechanics** - New ways to interact
- **New Depth** - Deeper systems
- **New Variety** - More options
- **New Goals** - New objectives

**Why**: Hooks maintain interest.

## How Cursor Uses Loop Evolution

### When Designing Loops

Cursor must:
1. **Design initial loop** - Simple, clear, engaging
2. **Plan evolution** - How loop will evolve
3. **Design evolution triggers** - When evolution happens
4. **Create evolution hooks** - What maintains interest

**Never design loops without evolution planning.**

---

### When Evaluating Designs

Cursor must ask:
- Is loop evolution planned?
- Are evolution triggers clear?
- Are evolution hooks present?
- Will evolution maintain engagement?

**Never accept designs without loop evolution.**

---

## Success Indicators

Loop evolution succeeds when:
- ✅ Loops evolve appropriately
- ✅ Evolution maintains engagement
- ✅ Triggers are clear
- ✅ Hooks are present
- ✅ Users progress through evolution
