# Prompt Hierarchy

## Purpose

How prompts relate to each other, enabling systematic prompt usage and chaining.

**Prompt hierarchy enables systematic workflow.**

---

## Hierarchy Structure

### Level 1: Master Prompt

**Purpose**: Core identity anchor.

**Prompt**: `master-design-prompt.md`

**When to Use**: Start of any project/feature.

**Why**: Establishes identity and mode.

---

### Level 2: Mode Prompts

**Purpose**: Activate specific modes.

**Prompts**:
- `research/` - Research prompts
- `ideation/` - Ideation prompts
- `comparison/` - Comparison prompts
- `design-synthesis/` - Design synthesis prompts
- `blueprint-generation/` - Blueprint prompts
- `implementation/` - Implementation prompts
- `auditing/` - Audit prompts
- `refactoring/` - Refactoring prompts
- `scaling/` - Scaling prompts

**When to Use**: When entering specific mode.

**Why**: Mode prompts guide workflow.

---

### Level 3: Task Prompts

**Purpose**: Specific task guidance.

**Prompts**: Specific prompts within each mode.

**When to Use**: When performing specific tasks.

**Why**: Task prompts provide detailed guidance.

---

## Prompt Chaining

### Research → Synthesis → Blueprint → Implementation

**Chain**:
1. Research prompts
2. Synthesis prompts
3. Blueprint prompts
4. Implementation prompts

**Why**: Chaining enables complete workflow.

---

## Integration Points

### Design Intelligence
- Prompts reference design layers
- Layers guide prompt usage
- Prompts enforce layers

### Knowledge Base
- Prompts reference knowledge
- Knowledge informs prompts
- Prompts accumulate knowledge

---

**This hierarchy enables systematic prompt usage and workflow management.**
