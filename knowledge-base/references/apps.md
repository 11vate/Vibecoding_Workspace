# App Design References

## Purpose

This document catalogs **notable apps** that demonstrate excellent design patterns, UX decisions, or interaction design. Each entry includes what it does well, what it does poorly, and what can be adapted.

---

## Entry Format

Each app entry follows this structure:

```markdown
## [App Name]

**Category**: [Category]
**Platform**: [Platforms]

**What it does well**:
- [Strength 1]
- [Strength 2]

**What it does poorly**:
- [Weakness 1]
- [Weakness 2]

**What can be adapted**:
- [Adaptable pattern 1]
- [Adaptable pattern 2]

**Notes**: [Additional observations]
```

## Productivity Apps

### Notion

**Category**: Productivity, Note-taking
**Platform**: Web, Desktop, Mobile

**What it does well**:
- Flexible content organization
- Powerful but accessible
- Clean, minimal UI
- Progressive disclosure

**What it does poorly**:
- Can be overwhelming
- Learning curve
- Performance issues with large docs
- Mobile experience limited

**What can be adapted**:
- Progressive disclosure patterns
- Flexible organization systems
- Clean, minimal UI design
- Powerful but accessible approach

**Notes**: Excellent example of powerful but accessible design.

---

### Todoist

**Category**: Productivity, Task Management
**Platform**: Web, Desktop, Mobile

**What it does well**:
- Simple, clear interface
- Quick task entry
- Effective filtering/organization
- Cross-platform sync

**What it does poorly**:
- Premium features gated
- Can be limiting for complex projects
- May not suit all workflows
- Limited collaboration

**What can be adapted**:
- Quick entry patterns
- Simple, clear interface design
- Effective filtering systems
- Cross-platform sync patterns

**Notes**: Excellent example of simple, effective task management.

## How to Use This Library

### When Designing UI/UX

1. **Search for similar apps**
2. **Review what they do well**
3. **Learn from their mistakes**
4. **Adapt patterns, don't copy**

### When Evaluating UX

1. **Compare to reference apps**
2. **Consider what works/doesn't work**
3. **Learn from failures**
4. **Adapt successful patterns**

### When Adding New Entries

1. **Follow entry format**
2. **Be specific about strengths/weaknesses**
3. **Note what can be adapted**
4. **Include relevant observations**
