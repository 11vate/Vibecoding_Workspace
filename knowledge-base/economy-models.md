# Economy Models Library

## Purpose

This library catalogs **economic system patterns** - how resources, currency, and value flow through a system. Each entry includes what it does, pros/cons, and when to use it.

---

## Entry Format

Each model entry follows this structure:

```markdown
## [Model Name]

**Description**: [What it does, how it works]

**Pros**: 
- [Strength 1]
- [Strength 2]

**Cons**: 
- [Weakness 1]
- [Weakness 2]

**Best Use Cases**: 
- [When to use 1]
- [When to use 2]

**Adaptation Notes**: [How to adapt, not copy]
```

## Currency Models

### Single Currency

**Description**: One currency type used for all transactions. Simple and straightforward.

**Pros**: 
- Simple and clear
- Easy to understand
- No conversion complexity
- Familiar pattern

**Cons**: 
- Limited monetization options
- May gate content simply
- Can become grind-heavy
- Less strategic depth

**Best Use Cases**: 
- Simple games
- Clear value systems
- Casual experiences
- Straightforward monetization

**Adaptation Notes**: 
- Balance earning vs. spending
- Provide multiple earning sources
- Avoid excessive grinding
- Make earning engaging

---

### Multi-Currency

**Description**: Multiple currency types (premium, earned, special) with different sources and uses. Enables complex monetization.

**Pros**: 
- Enables monetization
- Creates strategic decisions
- Provides variety
- Allows content gating

**Cons**: 
- Can be confusing
- May create pay-to-win
- Can feel manipulative
- Requires balancing

**Best Use Cases**: 
- Monetization systems
- Strategic resource management
- Complex economies
- Content gating

**Adaptation Notes**: 
- Keep currencies clear
- Avoid pay-to-win
- Provide conversion options
- Balance currency sources

---

## Resource Models

### Renewable Resources

**Description**: Resources that regenerate over time (energy, stamina, materials). Time-gated progression.

**Pros**: 
- Controls pacing
- Prevents overuse
- Creates strategic decisions
- Enables monetization

**Cons**: 
- Can frustrate if too limiting
- May gate content unnecessarily
- Can create FOMO
- May feel like work

**Best Use Cases**: 
- Pacing control
- Strategic decision-making
- Resource management
- Monetization systems

**Adaptation Notes**: 
- Balance regeneration rates
- Provide multiple resource types
- Make resource management engaging
- Avoid arbitrary gating

---

- Enables resource management
- Creates scarcity

**Cons**: 
- Can become grind-heavy
- May gate content
- Can create FOMO
- May feel like work

**Best Use Cases**: 
- Crafting systems
- Resource management
- Strategic decision-making
- Scarcity emphasis

**Adaptation Notes**: 
- Balance consumption rates
- Provide multiple resource sources
- Make resources meaningful
- Avoid excessive grinding

## Sink Models

### Upgrade Sinks

**Description**: Currency/resources spent on upgrades that provide permanent benefits. Long-term investment.

**Pros**: 
- Provides long-term goals
- Creates meaningful spending
- Rewards investment
- Enables progression

**Cons**: 
- Can become expensive
- May gate progression
- Can create power creep
- May require balancing

**Best Use Cases**: 
- Long-term progression
- Investment systems
- Meaningful spending
- Progression gating

**Adaptation Notes**: 
- Balance upgrade costs
- Make upgrades meaningful
- Provide multiple upgrade paths
- Avoid excessive costs

---

### Cosmetic Sinks

**Description**: Currency/resources spent on cosmetic items (skins, effects, visual customization). Expression-focused.

**Pros**: 
- Enables expression
- Doesn't affect gameplay
- Provides variety
- Enables monetization

**Cons**: 
- May not appeal to all players
- Can be expensive
- May feel like waste
- Limited value

**Best Use Cases**: 
- Expression systems
- Cosmetic customization
- Monetization (non-P2W)
- Variety emphasis

**Adaptation Notes**: 
- Balance prices
- Make cosmetics meaningful
- Provide variety
- Enable expression

---

### Rental/Subscription Sinks

**Description**: Recurring costs for temporary benefits (premium membership, time-limited bonuses, recurring access).

**Pros**:
- Creates recurring revenue
- Provides ongoing value
- Enables flexible pricing
- Reduces commitment anxiety

**Cons**:
- Can feel manipulative
- May create pressure
- Can exclude players
- Requires ongoing value

**Best Use Cases**:
- Live service games
- Ongoing benefits
- Premium features
- Recurring monetization

**Adaptation Notes**:
- Provide clear value
- Make cancellation easy
- Avoid predatory practices
- Balance free vs. paid

---

### Maintenance/Upkeep Sinks

**Description**: Ongoing costs to maintain assets, bases, or systems (repair costs, upkeep fees, maintenance).

**Pros**:
- Creates ongoing engagement
- Prevents hoarding
- Controls inflation
- Adds strategic depth

**Cons**:
- Can feel punishing
- May create frustration
- Can discourage investment
- May feel like work

**Best Use Cases**:
- Base building games
- Strategic resource management
- Inflation control
- Ongoing engagement

**Adaptation Notes**:

- Make upkeep engaging
- Provide multiple payment options
- Avoid excessive punishment

## Scarcity Models

### Limited Supply Economy

**Description**: Fixed or limited quantities of resources/items, creating scarcity and strategic competition.

**Pros**:
- Creates urgency
- Increases value
- Enables competition
- Provides exclusivity

**Cons**:
- Can frustrate players
- May create FOMO
- Can favor early adopters
- May exclude casual players

**Best Use Cases**:
- Competitive economies
- Collectible items
- Event exclusives
- Strategic scarcity

**Adaptation Notes**:
- Communicate scarcity clearly
- Provide catch-up mechanisms
- Avoid excessive FOMO
- Balance exclusivity

---

### Deflation/Inflation Control

**Description**: Economic mechanisms that control currency/resource value over time through sinks, sources, and balancing.

**Pros**:
- Maintains economy health
- Prevents exploitation
- Creates long-term stability
- Enables adjustments

**Cons**:
- Can be complex to implement
- May require constant monitoring
- Can affect player experience
- Difficult to balance

**Best Use Cases**:
- Long-term economies
- Player-driven markets
- Complex systems
- Live service games

**Adaptation Notes**:
- Monitor economy metrics
- Implement gradual changes
- Communicate adjustments
- Balance sources and sinks

## Lottery/Chance Models

### Gacha/Random Rewards

**Description**: Players spend currency for randomized rewards, creating excitement and collection drive.

**Pros**:
- Creates excitement
- Enables monetization
- Provides surprise
- Drives collection

**Cons**:
- Can feel exploitative
- May create gambling behavior
- Can be unfair
- Often controversial

**Best Use Cases**:
- Collection games
- Monetization (ethical only)
- Surprise mechanics
- Variety in rewards

**Adaptation Notes**:
- Be transparent about odds
- Implement pity systems
- Consider ethical implications
- Avoid predatory practices
