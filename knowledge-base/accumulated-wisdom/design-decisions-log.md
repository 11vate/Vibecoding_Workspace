# Design Decisions Log

## Purpose

This log accumulates significant design decisions across projects, enabling Cursor to learn from past decisions and maintain consistency.

**Documenting decisions creates institutional memory.**

---

## Decision Entry Format

```markdown
## [Decision Title]

**Date**: [Date]
**Project**: [Project name]
**Context**: [What problem this solved]

**Decision**: [What was decided]

**Alternatives Considered**:
- [Alternative 1]: [Why rejected]
- [Alternative 2]: [Why rejected]

**Rationale**: [Why this decision was made]

**Outcome**: [How it worked out]

**Lessons Learned**:
- [Lesson 1]
- [Lesson 2]

**Reusability**: [Can this be reused?]
```

## Usage Guidelines

### When to Document

**Document decisions when**:
- Significant design choice
- Multiple alternatives considered
- Non-obvious decision
- Reusable pattern

**Why**: Document important decisions.

---

### How to Reference

**Reference process**:
1. Search for similar decisions
2. Review rationale and outcome
3. Apply lessons learned
4. Maintain consistency or justify deviation

**Why**: Reference past decisions for consistency.
