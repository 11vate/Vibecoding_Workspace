# Pattern Matcher Tool

## Purpose

This tool matches patterns across projects, enabling Cursor to identify applicable patterns and suggest adaptations.

**Pattern matching enables knowledge reuse across projects.**

---

## How It Works

### Pattern Analysis

**Process**:
1. Analyze project structure
2. Identify pattern characteristics
3. Match against pattern library
4. Rank matches by relevance
5. Suggest adaptations

**Output**: Ranked pattern matches with adaptation suggestions

---

### Matching Criteria

**What to Match**:
- Problem similarity
- Context similarity
- Constraint similarity
- Goal similarity

**Why**: Multiple criteria ensure good matches.

---

## Usage

### When to Use

**Use pattern matcher when**:
- Starting new projects
- Solving similar problems
- Looking for proven solutions
- Avoiding reinvention

**Why**: Pattern matching finds applicable solutions.

---

### How to Use

**Usage process**:
1. Describe current problem/context
2. Run pattern matcher
3. Review matches
4. Assess applicability
5. Adapt and apply

**Why**: Systematic usage ensures good results.

---

## Integration Points

### Pattern Libraries
- Searches all pattern libraries
- Matches against patterns
- Suggests related patterns

### Knowledge Base
- References accumulated wisdom
- Links to pattern variants
- Connects to combinations
