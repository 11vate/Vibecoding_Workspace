# Insight Extractor Tool

## Purpose

This tool extracts insights from completed projects, enabling Cursor to learn from past work and accumulate knowledge.

**Insight extraction enables knowledge accumulation.**

---

## How It Works

### Extraction Process

**Process**:
1. Analyze project documentation
2. Identify key insights
3. Extract lessons learned
4. Categorize insights
5. Create knowledge entries

**Output**: Extracted insights ready for knowledge base

---

### Extraction Criteria

**What to Extract**:
- Successful patterns
- Failed patterns
- Design decisions
- Technical insights
- UX insights

**Why**: Multiple insight types enable comprehensive learning.

---

## Usage

### When to Use

**Use insight extractor when**:
- Completing projects
- Reviewing project outcomes
- Accumulating knowledge
- Learning from experience

**Why**: Extract insights while they're fresh.

---

### How to Use

**Usage process**:
1. Point to project documentation
2. Run insight extractor
3. Review extracted insights
4. Validate insights
5. Add to knowledge base

**Why**: Systematic extraction ensures quality.

---

## Integration Points

### Knowledge Base
- Adds to accumulated wisdom
- Updates pattern libraries
- Creates cross-project insights

### Design Intelligence
- Informs future decisions
- Guides pattern selection
- Supports reasoning

---

**This tool enables Cursor to extract and accumulate insights from completed projects.**
